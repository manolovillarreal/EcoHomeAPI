API REST + JWT + RBAC + PostgreSQL

EcoHome Store API

Backend academico de e-commerce construido con Node.js, Express y PostgreSQL, con autenticacion JWT, refresh tokens, control por roles y administracion de productos.

La API mantiene compatibilidad hacia atras en GET /products: sin query params devuelve el arreglo tradicional, y con paginacion devuelve { data, pagination }.

Descripcion general

La aplicacion implementa una arquitectura RESTful con persistencia en PostgreSQL, autenticacion con access token y refresh token, y RBAC con roles admin, staff y client.

Funcionalidades principales

  • Registro y autenticacion de usuarios con JWT
  • Refresh tokens persistidos en PostgreSQL
  • Gestion de catalogo de productos con CRUD
  • RBAC para roles admin, staff y client
  • Paginacion y filtro por creador en GET /products
  • Trazabilidad del creador del producto usando req.user.id
  • Metricas del usuario autenticado con conteo de productos creados

Endpoints principales

POST /auth/signup - Registro publico, siempre crea usuarios client
POST /auth/login - Devuelve accessToken y refreshToken
POST /auth/refresh - Emite un nuevo accessToken
GET /products - Lista productos; soporta page, limit y created_by
GET /products/:id - Obtener producto con creador
POST /products - Crear producto (admin, staff)
PATCH /products/:id - Actualizacion parcial (admin, staff)
PUT /products/:id - Actualizacion completa compatible (admin, staff)
DELETE /products/:id - Eliminar producto (admin)
GET /users/me/stats - Ver metricas del usuario autenticado (admin, staff)

Respuestas y roles

Producto con trazabilidad Incluye created_by y creator: { id, name } sin exponer datos sensibles.
Login y refresh POST /auth/login devuelve token, accessToken y refreshToken para compatibilidad.
Paginacion opcional Cuando se usan query params, GET /products responde con data y pagination.
RBAC Roles disponibles: admin, staff y client. Signup sigue creando usuarios client.

Documentacion y pruebas

La API cuenta con un archivo openapi.json, una coleccion de Postman y pruebas con Jest + Supertest. Puedes ejecutar npm test -- --verbose para ver los escenarios descriptivos en consola.
