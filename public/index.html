API REST + JWT + PostgreSQL

EcoHome Store API

Backend academico de e-commerce construido con Node.js, Express y PostgreSQL, enfocado en autenticacion con JWT, administracion de productos y trazabilidad por usuario.

Los productos ahora registran created_by automaticamente desde el JWT y las consultas incluyen un objeto creator con datos minimos del usuario.

Descripcion general

La aplicacion implementa una arquitectura RESTful con persistencia en PostgreSQL, autenticacion mediante JWT y control de roles para flujos administrativos.

Funcionalidades principales

  • Registro y autenticacion de usuarios
  • Gestion de catalogo de productos con CRUD
  • Trazabilidad del creador del producto usando req.user.id
  • Metricas del usuario autenticado con conteo de productos creados
  • Proteccion de rutas mediante JWT y control de roles

Endpoints principales

POST /auth/signup - Registro de usuario
POST /auth/login - Autenticacion
GET /products - Listar productos con creador
GET /products/:id - Obtener producto con creador
POST /products - Crear producto (admin, usa JWT para created_by)
PATCH /products/:id - Actualizar producto (admin)
DELETE /products/:id - Eliminar producto (admin)
GET /users/me/stats - Ver metricas del usuario autenticado

Respuestas nuevas

Producto con trazabilidad Incluye created_by y creator: { id, name } sin exponer datos sensibles.
Metricas del usuario GET /users/me/stats devuelve userId, name y productCount.

Documentacion y pruebas

La API cuenta con un archivo openapi.json para importar en Postman y una coleccion lista para validar autenticacion, productos y el flujo de metricas.
